Title: Core Architecture of Modern NPUs

Modern Neural Processing Units share a common architectural philosophy despite coming from competing vendors: maximize the throughput of multiply-accumulate (MAC) operations while minimizing data movement, because data movement consumes far more energy than computation in nanometer-scale CMOS processes.

At the heart of every NPU is a systolic array or a variant thereof. A systolic array is a grid of processing elements (PEs) where data flows rhythmically from one PE to the next, like blood pulsed through a heart. Each PE performs a MAC operation: it multiplies two inputs and accumulates the result into a running sum. Google's TPU v1 contained a 256x256 systolic array of 8-bit integer MACs, yielding 65,536 MAC operations per clock cycle. At 700 MHz, this delivered approximately 92 tera-operations per second (TOPS) of 8-bit integer throughput.

The systolic array architecture offers a crucial advantage: weights are loaded once and reused across many input activations, dramatically reducing memory bandwidth pressure. This property is called weight stationary dataflow. Alternative dataflows include output stationary, where partial sums accumulate in place, and row stationary, used in MIT's Eyeriss research chip, which minimizes all three types of data movement simultaneously.

On-chip memory hierarchy is as important as the compute array itself. NPUs typically feature multiple levels of scratchpad SRAM, sometimes called global buffers or activation buffers, that feed the systolic array. Unlike caches, which rely on hardware-managed associativity, scratchpad memories are software-managed, allowing compilers to precisely schedule data movement. The MIT Eyeriss chip used a 108 KB global buffer alongside 12 KB of PE-local memory. Apple's M2 Neural Engine, by contrast, deploys a much larger on-chip memory to support larger transformer model workloads.

Quantization support is a defining feature of NPU design. Neural networks can tolerate reduced numerical precision without significant accuracy loss, so NPUs implement INT8, INT4, and increasingly INT2 arithmetic alongside standard FP16 and BF16 floating point. The ratio of integer to floating-point throughput is a key differentiator: Qualcomm's Hexagon NPU in the Snapdragon 8 Gen 3 achieves 45 TOPS of INT4 performance, enabling on-device large language model inference with models up to 10 billion parameters.

Dataflow scheduling, the compiler's job of mapping neural network operations onto the physical array, is arguably as important as the hardware itself. Companies like Google with XLA, Apple with Core ML, and Qualcomm with the AI Model Efficiency Toolkit (AIMET) invest heavily in compiler toolchains that extract maximum utilization from their NPU hardware. Poorly compiled models can achieve less than 20% of peak hardware throughput, while well-optimized models approach 80-90% utilization.

Interconnect fabric between the NPU, CPU, GPU, and memory subsystem strongly influences system-level performance. Apple's Unified Memory Architecture (UMA), introduced with the M1 chip in 2020, allows the Neural Engine, CPU, and GPU to share the same high-bandwidth LPDDR5 memory pool without copying data across discrete buses. This eliminates a major bottleneck in heterogeneous AI workloads and became a template that competitors subsequently adopted.
